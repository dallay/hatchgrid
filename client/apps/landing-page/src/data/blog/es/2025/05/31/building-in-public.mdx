El viaje de construir Hatchgrid comenzó con una frustración clara: los creadores individuales utilizan demasiadas herramientas para hacer una sola cosa — entregar contenido de forma constante. Ya sea gestionando una newsletter, vendiendo un producto o publicando en redes, el proceso está fragmentado. Hatchgrid quiere cambiar eso centralizando y automatizando el flujo de trabajo para creadores, comenzando por el correo.

## ¿Por qué empezar por el correo?

El correo es el canal más directo y propio que tiene un creador. Es donde se construye confianza y se genera ingresos. Por eso nuestro MVP se enfoca en newsletters: redactarlas, hacer crecer la lista de suscriptores, automatizar correos de bienvenida y, más adelante, permitir monetización con niveles pagos.

## Funcionalidades principales del MVP

- Editor de newsletters (Markdown/WYSIWYG)
- Gestión de suscriptores con etiquetas
- Formularios embebibles para captar leads
- Automatización de correos de bienvenida
- Analíticas básicas de apertura y clics
- Integración con Stripe (opcional en la primera fase)

## Stack tecnológico

Usamos **Astro** para la landing (rendimiento y SEO), **Vue.js** para la interfaz de la app y **Spring Boot con Kotlin** para el backend — un stack alineado con velocidad y escalabilidad. También consideramos Supabase para autenticación y almacenamiento cuando sea útil.

## Filosofía de diseño

Creemos en la simplicidad, velocidad y confianza. Eso significa UX limpio, configuración mínima y cero funciones infladas. Cada línea del MVP responde a una sola pregunta: *¿esto ayuda directamente a un creador a publicar y crecer más rápido?*

## Qué sigue

El MVP será probado por adoptantes tempranos, cuyo feedback dará forma a las próximas versiones. Las metas futuras incluyen publicación multicanal, programación de contenido y segmentación avanzada.

Este blog será nuestro diario de construcción — transparente, honesto y frecuente. Si eres creador, constructor o alguien que valora la autonomía digital, te invitamos a seguir el proceso.
