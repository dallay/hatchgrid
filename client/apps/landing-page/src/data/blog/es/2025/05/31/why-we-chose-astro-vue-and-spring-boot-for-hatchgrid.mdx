Elegir la tecnología adecuada para un producto no se trata solo de seguir tendencias. En Hatchgrid, buscamos equilibrio entre velocidad, escalabilidad y dominio del stack.

Elegimos **Astro** para la landing pública por su rendimiento y ventajas SEO. Es rápido, sin JS por defecto, y se integra perfecto con Tailwind.

**Vue** potencia la interfaz de nuestra app. Es flexible y su curva de aprendizaje es suave, ideal para equipos pequeños. La Composition API de Vue 3 nos da estructura para crecer.

**Spring Boot con Kotlin** es nuestro backend. Kotlin nos da seguridad de tipos y claridad, mientras que Spring nos ofrece herramientas robustas para seguridad, base de datos y modularidad.

Estamos considerando Supabase para auth o almacenamiento ligero, pero preferimos mantener la lógica principal en Spring Boot para tener control y consistencia.

Este stack nos permite movernos rápido, con seguridad, y escalar a medida que crece nuestra base de usuarios.
